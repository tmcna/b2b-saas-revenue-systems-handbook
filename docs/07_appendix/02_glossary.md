# 付録：用語集

本書に登場する主な用語を、英語略称・日本語訳・定義の順に整理したものです。各用語の詳細は本文中の初出箇所を参照してください。

---

## 業務・収益指標

| 英語 | 日本語 | 定義 |
|------|--------|------|
| ARR (Annual Recurring Revenue) | 年間経常収益 | サブスクリプション収益を年換算した金額。MRR × 12 で計算することが多い |
| MRR (Monthly Recurring Revenue) | 月次経常収益 | 月単位で計上される経常的なサブスクリプション収益 |
| NRR (Net Revenue Retention) | 純収益維持率 | 既存顧客からの収益が前期比でどれだけ維持・拡大したかを示す指標。Expansion を含む |
| GRR (Gross Revenue Retention) | 粗収益維持率 | Churn と Contraction のみを考慮した収益維持率。Expansion は含まない |
| Expansion | 拡張収益 | 既存顧客からのアップセル・クロスセル・シート追加による収益増加 |
| Contraction | 縮小収益 | 既存顧客のダウングレード・シート削減による収益減少 |
| Churn | 解約 | 顧客またはサブスクリプションの解約。収益 Churn（金額）と顧客 Churn（件数）で区別することが多い |
| Cohort | コホート | 同一期間（例：同じ月）に獲得した顧客グループ。継続率の時系列分析に使う |
| Bookings | 受注額 | 契約締結時点で確定した将来の収益総額。未請求・未計上の分も含む |

## 業務プロセス

| 英語 | 日本語 | 定義 |
|------|--------|------|
| Lead-to-Cash | リードから入金まで | リード獲得から入金完了までの業務プロセス全体。最も広義のフロー |
| Quote-to-Cash | 見積から入金まで | 見積作成から入金完了までのプロセス。Lead-to-Cash の部分集合 |
| Order-to-Cash | 受注から入金まで | 注文受付から入金完了までのプロセス。Quote-to-Cash の部分集合 |
| SLG (Sales-Led Growth) | 営業主導の成長 | 営業担当者が商談・提案・クローズを行う成長モデル |
| PLG (Product-Led Growth) | プロダクト主導の成長 | プロダクト自体が獲得・活性化・拡大を担う成長モデル |
| RevOps (Revenue Operations) | 収益オペレーション | 営業・マーケティング・CS を横断してデータと業務プロセスを統合する機能 |
| ICP (Ideal Customer Profile) | 理想顧客プロファイル | 最も価値を提供しやすく収益性が高い顧客像の定義 |
| SOW (Statement of Work) | 業務範囲記述書 | プロフェッショナルサービス契約で成果物・工数・期間を定める書面 |

## システム・アーキテクチャ

| 英語 | 日本語 | 定義 |
|------|--------|------|
| System of Record (SoR) | 正本 | あるデータの「唯一の正しい情報源」として扱うシステム。本書では「正本」と表記する |
| Master Data | マスターデータ | 複数の業務プロセスから参照される、変化の少ない基準情報（顧客・商品・価格表など） |
| MDM (Master Data Management) | マスターデータ管理 | 複数システムに分散したマスターデータを統一的に管理する取り組み |
| Bounded Context | 境界づけられたコンテキスト | ある言葉や概念が「同じ意味で通用する範囲」。ドメイン駆動設計（DDD）の概念 |
| Append-only | 追記型 | データを変更・削除せず追記のみで管理する設計。総勘定元帳や入金レコードで採用される |
| Event-Driven | イベント駆動 | 業務上の出来事（イベント）を起点に後続の処理を呼び出す設計パターン |
| CPQ (Configure Price Quote) | 見積・価格管理システム | 商品の組み合わせ・価格設定・見積書作成を管理するシステム |
| CRM (Customer Relationship Management) | 顧客関係管理システム | 顧客情報・商談・活動履歴を管理するシステム |
| ERP (Enterprise Resource Planning) | 基幹業務システム | 財務・会計・調達など企業の基幹業務を統合管理するシステム |
| DWH (Data Warehouse) | データウェアハウス | 複数システムのデータを集約・蓄積し、分析クエリに特化したデータ基盤 |

## 会計・財務

| 英語 | 日本語 | 定義 |
|------|--------|------|
| Revenue Recognition | 収益認識 | 売上をいつ・いくら計上するかの会計上のルール。SaaSでは IFRS 15 / ASC 606 が基準 |
| Deferred Revenue | 前受収益 | 入金済みだが未だ役務提供が完了していない収益。バランスシート上の負債として計上される |
| IFRS (International Financial Reporting Standards) | 国際財務報告基準 | IASB が策定する国際的な会計基準。収益認識は IFRS 15 が対象 |
| GAAP (Generally Accepted Accounting Principles) | 一般に認められた会計原則 | 米国の会計基準（US-GAAP）。収益認識は ASC 606 が対象 |
| AR Aging | 売掛金エイジング | 未回収の売掛金を滞留日数（例：30日・60日・90日超）別に分類した分析 |
| General Ledger (GL) | 総勘定元帳 | すべての財務取引を勘定科目別に集積した会計記録の台帳 |
| Credit Note | クレジットノート | 発行済み請求書の金額を減額するための取消文書。請求書そのものは変更しない |
