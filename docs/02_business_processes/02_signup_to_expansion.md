# 【プロセス編 2】PLGのSignup-to-Expansion——プロダクトが営業する世界

SLGでは、営業担当者が見込み客に電話をかけ、見積を送り、契約を結びます。PLGでは、それらのステップのほとんどが存在しません。ユーザーが自分でサインアップし、プロダクトを使い、価値を感じたときに自分でクレジットカードを登録します。プロダクト自体が、獲得・育成・転換を担うのです。

---

## 全体の流れ

```
[ユーザーによるセルフサインアップ]
サインアップ（Sign Up）
  ↓
アクティベーション（Activation）
  ↓ 価値を体験できたか？
  ↓ NO → 離脱（Churn）
  ↓ YES
エンゲージメント（Engagement）
  ↓ 転換のトリガー
  ├── 機能制限に到達 ──────────→ 有料転換（Conversion）
  ├── チーム招待が増加 ─────────→ 有料転換（Conversion）
  └── 営業からのアプローチ ─────→ エンタープライズ商談（PLS）→ SLGフローへ
        ↓
[有料化]
サブスクリプション開始（Subscription）
  ↓
請求（Invoice）→ 入金（Payment）
  ↓
拡張のきっかけ
  ├── ユーザー数が増える ─────→ アップグレード（Expansion）→ サブスクリプションへ戻る
  └── 上位プランの機能が必要 ──→ アップグレード（Expansion）→ サブスクリプションへ戻る
        ↓
継続か解約か
  ├── 解約 → 離脱
  └── 継続 → エンゲージメントへ戻る
```

---

## 各ステップの説明

### 1. サインアップ（Sign Up）

ユーザーが自分でアカウントを作成します。営業担当者は介在しません。

- メールアドレスのみで登録できるケース
- GoogleやGitHubのアカウントで即時ログインできるケース（ソーシャルログイン）
- クレジットカード不要でスタートできるケース

サインアップの摩擦（手間）を減らすことが、PLGの重要な設計原則です。

### 2. アクティベーション（Activation）

ユーザーがプロダクトの価値を初めて体験する瞬間です。「**アハモーメント（Aha Moment）**」とも呼ばれます。アクティベーションが達成されないと、ユーザーはそのまま離脱します。

アクティベーションの定義はプロダクトによって異なります。

| プロダクトの例 | アクティベーションの定義 |
|-------------|----------------------|
| タスク管理ツール | 最初のタスクを作成・完了した |
| コミュニケーションツール | チームメンバーを招待してメッセージを送った |
| データ分析ツール | 最初のダッシュボードを作成した |

### 3. エンゲージメント（Engagement）

ユーザーがプロダクトを継続的に使い、習慣化した状態です。エンゲージメントの高いユーザーは、有料転換・チーム拡大・長期継続の可能性が高いです。プロダクト利用データを分析することで、エンゲージメントの高低を把握します。

### 4. 有料転換（Conversion）

無料ユーザーが有料プランに移行するステップです。PLGの収益の起点です。転換のトリガーには主に2種類あります。

**機能制限への到達（Feature Gate）**
無料プランで使える機能・容量・人数の上限に達したとき、有料プランへの案内が表示されます。

**プロダクト内のアップグレード促進**
使用頻度が高い機能に有料バッジを表示したり、「チームで使うにはアップグレードを」といったメッセージを出したりします。

### 5. サブスクリプション・請求・入金

有料転換後は、クレジットカードによる自動課金が始まります。SLGと異なり、**書面による見積・交渉・社内承認プロセスは発生しません**。ユーザーがプロダクト内でプランを選択し、カード情報を入力するだけで完了します。なお、ユーザーが利用規約に同意してサービスを開始する行為は民法上の契約（申込と承諾）が成立しており、「契約が発生しない」という意味ではありません。

### 6. 拡張（Expansion）

既存の有料ユーザーが、さらに多くの費用を支払うようになるステップです。

| 拡張のパターン | 説明 |
|-------------|------|
| シート追加 | チームメンバーが増えてライセンスを追加します |
| アップグレード | 上位プランに移行します |
| アドオン | 追加機能を購入します |

PLGにおける拡張収益（Expansion Revenue）は、新規獲得コストがかからないため、収益性が高いです。

### 7. PLS（Product-Led Sales）への移行

個人・チームの利用が広がると、組織全体への展開を目指す営業活動が始まります。これを **PLS（Product-Led Sales、プロダクト主導の営業）** と呼びます。

```
個人・チームの利用データを営業がモニタリング
  ↓ 使用量・チーム規模が閾値を超えると
営業がアプローチ（インバウンドセールス）
  ↓
SLGのフロー（見積→契約→エンタープライズ請求）へ
```

## SLGプロセスとの違い

| | SLG（Lead-to-Cash） | PLG（Signup-to-Expansion） |
|--|---------------------|--------------------------|
| 獲得の起点 | 営業・マーケティング | プロダクト（セルフサインアップ） |
| 見積・契約 | 必須 | 不要（小〜中規模） |
| 転換の判断者 | 会社の意思決定者 | ユーザー自身 |
| 請求方式 | 請求書払いが多い | クレジットカード自動課金 |
| 拡張の主な手段 | 営業によるアップセル | プロダクト内の自然な拡大 |

---

## このプロセスで使われるシステム

| ステップ | 担うシステムの例 |
|---------|--------------|
| サインアップ・認証 | 自社プロダクト / Auth0 / Cognito |
| プロダクト利用データの収集 | Segment / Mixpanel / Amplitude |
| アクティベーション・エンゲージメント促進 | Intercom / Customer.io |
| 決済・サブスクリプション管理 | Stripe |
| PLS（営業介入） | HubSpot / Salesforce |

---

## 重要な指標

| 指標 | 説明 |
|------|------|
| アクティベーション率 | サインアップしたユーザーのうち、アクティベーションを達成した割合 |
| コンバージョン率 | 無料ユーザーのうち、有料に転換した割合 |
| PQL（Product Qualified Lead） | プロダクトの利用状況から、有料転換の可能性が高いと判断されたユーザー |
| タイム・トゥ・バリュー（TTV） | サインアップからアクティベーションまでにかかる時間 |
| 拡張収益率（NRR） | 既存顧客からの収益が前期比でどれだけ増減したか |
