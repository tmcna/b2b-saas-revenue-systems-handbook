# 各ドメインの責務と設計——なぜこの境界で分かれるのか

第1章で手に入れた型分類は、業務エンティティの性質を読み解くためのレンズです。本章ではそのレンズを使って、B2B SaaSの収益系システムに登場する12のドメインを一つひとつ見ていきます。

各ドメインは「何ができるか」ではなく「なぜ存在するか」から説明します。この問いに答えることで、特定のツールの使い方ではなく、設計判断の根拠が見えてきます。ツールが変わっても、ドメインが存在する理由は変わらないからです。

12のドメインは多く見えますが、すべてLead-to-Cashのどこかに位置しています。1つのドメインを読むたびに、収益の流れ全体の中でその位置が見えてくるはずです。
