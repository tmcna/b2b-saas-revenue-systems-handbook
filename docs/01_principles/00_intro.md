# 業務オブジェクトの型分類——設計判断を導くための語彙

第0章では「なぜ収益系システムは統合できないのか」を理解しました。しかし、原因がわかっても、次の問いが残ります。「では、どう設計すればいいのか」。

設計の場面では、具体的な判断を迫られます。見積書と請求書は同じシステムで管理できるか。契約書が変更されたとき、下流のシステムに何を伝えるべきか。入金データはなぜ上書きではなく追記で管理しなければならないのか——こうした判断を、その都度ゼロから考えていては一貫性が保てません。

本章では、これらの判断を導くための枠組みを提供します。業務オブジェクトには、表面上の種類の多さとは別に、性質を決定づける「型」があります。型を理解すると、見慣れないオブジェクトが登場したときでも、その性質と設計上の扱いを導き出せるようになります。
