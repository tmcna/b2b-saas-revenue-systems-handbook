# ビジネスモデル別の構成パターン——原理を実装に変える

第0〜2章は「原理」でした。業務エンティティの型・各ドメインの責務・業務プロセスの流れ——これらはビジネスモデルや規模に関係なく成立する構造です。

本章では問いが変わります。「自社では、どのシステムをどう組み合わせるべきか」。

この問いに唯一の正解はありません。ビジネスモデルが違えば、データの流れの起点が変わります。規模が変われば、手動で済むことと自動化が必要なことの境界が変わります。本章は、その判断を具体的なパターンとして示します。特定ツールの選定ガイドではなく、「この構造ならこの構成が合理的」という設計の根拠を提供します。
