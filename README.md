# B2B SaaS Revenue Systems Handbook

顧客管理・請求・会計のシステムのデータモデルを統合できないのは、ツールや組織の問題ではなく構造的な必然です。この前提に立ち、B2B SaaS の収益系システムを正しく設計するための体系を提供します。

中心となるのは、業務オブジェクトを4つの型（交渉・コミットメント・履行・決済）に分類する枠組みです。この分類を理解すると、見積・契約・請求書・入金などの性質——可変か不変か、どのシステムが正本を持つべきか——を導出できます。各ドメインの責務と、ビジネスモデル・成長フェーズに対応したシステム構成パターンはすべて、この型分類から導かれます。

対象は B2B SaaS（SLGとPLGの両方）を中心に扱います。HubSpot・Salesforce・Stripe などは例として登場しますが、特定ツールの使い方ではなく業務の構造とその設計判断が中心です。

## 読み方

**第0章** から始めてください。「なぜ顧客管理・請求・会計のデータモデルは統合できないのか」を理解すると、残りの内容がすべてその帰結として読めます。

**第Ⅰ部（原理）** では、業務オブジェクトの型分類と基礎概念を扱います。型分類を理解すると、任意の業務オブジェクトの性質を導出できます。

**第Ⅱ部（ドメイン）** では各ドメインを「なぜ存在するか」から説明します。型分類の文脈で読むと理解が深まります。

**第Ⅲ部（業務プロセス）** ではドメインが実際の業務フローの中でどう動くかを説明します。

**第Ⅳ部（システム構成パターン）** は特定のツール群ではなく、ビジネスモデルと成長フェーズに対応したパターンとして説明します。

**第Ⅴ部（データ分析）** は業務システムのデータを分析基盤に取り込む際のデータモデル設計を扱います。
